NexAnime

AnimePahe-inspired anime streaming site powered by the AniList GraphQL API. Search, browse by genre, filter by season — full discovery experience without a backend.

AniList GraphQL
Vanilla JS

How I Made It

AniList uses GraphQL which was new to me at the time. I expected it to be more complicated than REST, but it's actually better for this use case — you request exactly the fields you need, so the response is lean and the UI stays fast even on slower connections.

javascript 
async function fetchSeasonal(season, year) {
  const query = `
    query ($season: MediaSeason, $year: Int) {
      Page(perPage: 24) {
        media(season: $season, seasonYear: $year,
              type: ANIME, sort: POPULARITY_DESC) {
          id
          title { romaji english }
          coverImage { large }
          episodes
          averageScore
          genres
        }
      }
    }
  `;

  const res = await fetch('https://graphql.anilist.co', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ query, variables: { season, year } })
  });

  const { data } = await res.json();
  return data.Page.media;
}

AniList has a rate limit of 90 requests per minute. I hit it on day one during testing by refreshing too fast. The fix was simple — cache seasonal data in sessionStorage so repeated visits to the same season page don't re-hit the API.

javascript 
async function getSeasonCached(season, year) {
  const key  = `season_${season}_${year}`;
  const hit  = sessionStorage.getItem(key);
  if (hit) return JSON.parse(hit);

  const data = await fetchSeasonal(season, year);
  sessionStorage.setItem(key, JSON.stringify(data));
  return data;
}

The genre filter is entirely client-side — once you've loaded a season's worth of data, filtering by genre is just .filter() on the cached array. No extra API call needed.
