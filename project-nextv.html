NexTV

Anime streaming platform with full TMDB integration. Browse trending shows, search any title, and watch — all from a clean dark interface with no backend.

TMDB API
Vanilla JS
CSS3

How I Made It

NexTV is fully client-side — no backend, no hosting costs beyond a static CDN. All data comes from the TMDB API fetched directly in the browser. The tricky constraint was rate limiting: TMDB allows 40 requests per 10 seconds, which sounds like a lot until you're loading a page with 20 posters and triggering 5 other calls at the same time.

javascript 
const BASE  = 'https://api.themoviedb.org/3';
const TOKEN = 'Bearer ' + TMDB_TOKEN;

async function fetchTrending(type = 'tv', page = 1) {
  const res = await fetch(
    `${BASE}/trending/${type}/week?page=${page}`,
    { headers: { Authorization: TOKEN } }
  );
  if (!res.ok) throw new Error(`TMDB ${res.status}`);
  return res.json();
}

// Debounced search — don't hammer the API on every keystroke
let timer;
searchInput.addEventListener('input', e => {
  clearTimeout(timer);
  timer = setTimeout(() => search(e.target.value), 400);
});

Search uses a 400ms debounce. Poster images are lazy-loaded with IntersectionObserver so the page doesn't try to load 200 images at once. Each image only gets its src set when it's about to scroll into view.

javascript 
// Lazy load poster images
const imgObserver = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      imgObserver.unobserve(img);
    }
  });
}, { rootMargin: '200px' });

document.querySelectorAll('img[data-src]')
  .forEach(img => imgObserver.observe(img));

The biggest thing I learned here: client-side APIs are fine for a solo project, but you're exposing your API key to anyone who opens DevTools. For a real product you'd proxy through a backend. For NexTV, the TMDB key is rate-limited and has no billing attached, so it's an acceptable tradeoff.
