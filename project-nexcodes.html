How I Made It

The idea is simple: Claude outputs the entire website as a single HTML file with embedded CSS and JS. The system prompt is strict — no explanations, no markdown fences, just raw HTML that the browser can run. Getting that consistency took a lot of iteration.

javascript 
async function generateSite(description) {
  const system = `You are a world-class frontend developer.
Output ONLY a complete, valid HTML document.
No explanations. No markdown. No code fences.
Embed all CSS in a <style> tag. Embed all JS in a <script> tag.
Dark theme, modern, professional.`;

  const res = await client.messages.create({
    model:      'claude-opus-4-5',
    max_tokens: 4096,
    system,
    messages: [{ role: 'user', content: description }]
  });

  return res.content[0].text; // raw HTML string
}

// Render in sandboxed iframe — safe, no XSS risk to parent
previewFrame.srcdoc = await generateSite(userInput.value);

The preview renders in a sandboxed iframe using srcdoc. This keeps the generated code fully isolated from the parent page — any JS the model writes can't touch the NexCodes UI.

Iteration works by passing the previous HTML back as context. The user says "make the header bigger" and the prompt sends both the original description and the last generated HTML, so Claude knows what it's modifying.

javascript 
// Iterative editing — pass previous HTML as context
async function updateSite(change, previousHtml) {
  return client.messages.create({
    model:      'claude-opus-4-5',
    max_tokens: 4096,
    system:     'Output ONLY valid HTML. No markdown. No explanation.',
    messages: [
      { role: 'user',      content: `Here is the current site:\n\n${previousHtml}` },
      { role: 'assistant', content: previousHtml },
      { role: 'user',      content: `Apply this change: ${change}` }
    ]
  }).then(r => r.content[0].text);
}

The main limitation is token budget. Large generated sites can hit the output limit, cutting the HTML mid-tag and breaking the render. I added a simple check that validates the HTML ends with </html> before updating the preview.
