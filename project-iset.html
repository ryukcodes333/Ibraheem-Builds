Iset

AI-powered goal roadmap app. Enter a goal and a timeframe, get a structured milestone plan with tasks and resources — all generated by Claude, saved to Supabase.

Node.js
Supabase
Claude API

How I Made It

The core challenge was prompt engineering. My first attempts returned vague, generic plans. The fix was constraining the output format strictly — Claude must return a JSON array of milestones with a specific schema. No prose, no markdown, just parseable JSON the UI can consume directly.

javascript 
const Anthropic = require('@anthropic-ai/sdk');
const client = new Anthropic();

async function generateRoadmap(goal, weeks) {
  const msg = await client.messages.create({
    model: 'claude-opus-4-5',
    max_tokens: 2048,
    messages: [{
      role: 'user',
      content: `Return ONLY valid JSON. No explanation. No markdown.

Goal: "${goal}"
Timeframe: ${weeks} weeks

Schema:
{"milestones":[{"week":1,"title":"string","tasks":["string"],"resources":["url"]}]}`
    }]
  });

  return JSON.parse(msg.content[0].text);
}

Supabase handled auth and storage. Row Level Security meant I didn't need to write any auth middleware — Supabase enforces that each user can only read their own saved roadmaps based on the JWT in their session token.

javascript 
// Save roadmap — RLS handles user scoping automatically
const { error } = await supabase
  .from('roadmaps')
  .insert({
    user_id: session.user.id,
    goal,
    weeks,
    milestones: roadmap.milestones,
    created_at: new Date().toISOString()
  });

if (error) throw error;

The main thing I'd do differently: add streaming. Right now the app waits for Claude to finish the full response before rendering anything, which can take 5–8 seconds. Streaming the response token by token would make it feel much faster even if the total time is the same.
